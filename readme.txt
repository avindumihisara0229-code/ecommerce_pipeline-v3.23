PROJECT OVERVIEW
This project contains an end-to-end local data pipeline built for the CM2606 Data Engineering module. The pipeline extracts raw e-commerce CSV data, stages it in a local Data Lake, and uses Apache Spark to perform ETL processing. The final output is a Star Schema Data Warehouse.

FILE AND FOLDER EXPLANATIONS

1. PYTHON SCRIPTS
- ingestion.py: The extraction script. It moves raw CSV files into the staging area.
- etl_pyspark.py: The core processing engine. It reads raw data using Spark, applies data cleansing, derives new metrics, enforces validation rules, and loads data into the Data Warehouse.
- orchestrator.py: The master execution script. It runs the ingestion flow first, followed by the ETL flow.
- config.json: Stores file paths and configurations.

2. STORAGE DIRECTORIES
- data_lake/raw/ : The staging area. Acts as a landing zone for the raw CSV files.
- data_warehouse/ : The final analytical sink. Contains the highly structured CSV files forming the Star Schema (fact_orders, dim_customer, dim_product).

3. MONITORING
- ecommerce_pipeline.log: Generated by orchestrator.py. Tracks pipeline executions.

HOW TO EXECUTE THE PIPELINE
To run the entire pipeline, navigate to the folder in your terminal and run:
python orchestrator.py
